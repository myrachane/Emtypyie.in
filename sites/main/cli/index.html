quick start

From zero to running

Four steps, in order — this is the path every new install follows

01

Install the package

Requires Node.js 18 or newer. This puts the emtypyie binary on your PATH.

npm install -g emtypyie
02

Launch the shell

This drops you into the emtypyie prompt, where every command below is typed with a leading /.

emtypyie emtypyie v2.4.1 — type /help to see what's here
03

Grab a project

Pulls a project from the registry and installs it locally. Pin a version with @version if you need to.

/get dosa installed dosa@1.9.0 → ~/.emtypyie/projects/dosa
04

Run it

Any installed project runs by name. Pass its own flags straight through after the name.

/dosa → dosa running on localhost:4200

commands

Command Reference

All commands use the / prefix. Arguments in <angle brackets> are required, [square brackets] are optional.

/help
List all available commands, or get details on one.
usage /help [command]
example /help wrap
/get <project>
Download and install a project from the registry.
usage /get <project>[@version]
example /get dosa@1.8.0
/bakafetch
Display system info with ASCII art, like neofetch.
usage /bakafetch
example /bakafetch
/wrap all <theme>
Change the CLI's color scheme to a built-in theme.
usage /wrap all <slate|green|amber|violet|cyan>
example /wrap all violet
/wrap bakafetch <color>
Change just the bakafetch accent color.
usage /wrap bakafetch <hex|name>
example /wrap bakafetch #22c55e
/clear
Clear the terminal screen.
usage /clear
example /clear
/list
List installed projects with their versions.
usage /list
example /list
/update
Update emtypyie itself to the latest version.
usage /update
example /update
/docs <project>
Open a project's documentation on GitHub.
usage /docs <project>
example /docs wiki
/changelog
View emtypyie's release history on GitHub.
usage /changelog
example /changelog
/<project>
Run an installed project by name, passing through any flags it accepts.
usage /<project> [args]
example /dosa --port 4200
/list <project>
Show detailed info for a single installed project.
usage /list <project>
example /list dosa

configuration

Configuring emtypyie

Settings persist across sessions in a single config file

emtypyie reads ~/.emtypyie/config.json on startup. Anything you set with /wrap gets written back here automatically, so you rarely need to edit it by hand — but it's plain JSON if you want to.

{ "theme": "amber", "bakafetchColor": "#f59e0b", "installDir": "~/.emtypyie/projects", "autoUpdate": true, "registry": "https://registry.emtypyie.in" }
Environment variableDescriptionDefault
EMTYPYIE_HOME Overrides where config and projects are stored. ~/.emtypyie
EMTYPYIE_THEME Sets the theme at launch, before any saved config loads. slate
EMTYPYIE_NO_COLOR Disables colored output for scripting or CI logs. unset
EMTYPYIE_REGISTRY Points /get at a custom project registry. registry.emtypyie.in

faq

Questions people actually ask

What is emtypyie, actually?+

A small terminal runtime for pulling down and running personal projects by name, instead of cloning repos and remembering how to start each one. Think of it as a personal package manager with a shell attached.

Does it work on Windows?+

Yes, through npm like any other global package. It's tested mainly on macOS and Linux, so if something looks off on Windows, open a GitHub issue with your terminal and Node version.

Where do installed projects live?+

Under ~/.emtypyie/projects by default, or wherever installDir in your config points. /list shows exactly what's installed and where.

How do I uninstall it?+

Run npm uninstall -g emtypyie, then remove ~/.emtypyie if you also want to clear installed projects and saved config.

Can I add my own theme?+

Not yet as a named theme — /wrap only ships the five built-ins. You can get close by combining a base theme with /wrap bakafetch for a custom accent.

Is this an official or commercial product?+

No — it's a personal project. See the disclaimer in the footer for the full picture.
