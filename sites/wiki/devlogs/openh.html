§1 Overview

OpenNH-Download is a Chrome extension (Manifest v3) that automates the tedious process of saving a full gallery. Open any reader page, click the extension icon, and hit Download — it handles everything else.

Core features:

  • One-click ZIP download — entire gallery packed and saved automatically
  • Auto-loops from page 1 to the end, stopping at the first 404
  • CDN-aware — reads i1, i2, i3… subdomains directly from the DOM, never guesses
  • Filenames padded: 0001.jpg, 0002.jpg… for correct sort ordering
  • ZIP named after gallery code — e.g. 177013.zip
  • Cancel anytime mid-download with no partial file saved
  • Zero external dependencies — built-in CRC-32 ZIP engine
§2 Why I built this

Manually right-clicking and saving images one at a time is genuinely painful for anything over ten pages. Existing tools either require Python scripts, sketchy executables, or browser plugins riddled with ads and tracking. I wanted something that lived in the browser, needed no install beyond "Load Unpacked", and just worked.

The CDN problem was the interesting bit. nhentai doesn't serve images from the main domain — they fan out across numbered subdomains that vary per gallery and aren't predictable from the URL alone. Getting that right required reading the actual page DOM instead of guessing. That ended up being the whole trick.

Also an excuse to write a ZIP encoder from scratch. Turns out the ZIP spec is not that scary when you only need Store mode and no compression.

§3 Architecture / Design

Three files do all the work. manifest.json declares the extension with MV3 permissions. zipper.js is a self-contained ZIP builder: it handles local headers, central directory, end-of-central-directory, and a full CRC-32 lookup table — no dependencies. popup.js wires it all together: injects scrapeImageSrc() into the active tab, parses the CDN URL, runs the fetch loop, and triggers the download.

popup.js — scrapeImageSrc() injected into the page
javascript 
function scrapeImageSrc() {
  const selectors = [
    '#image-container img',
    '.image-container img',
    'section#image-container img',
    '.reader-images img',
  ];
  for (const sel of selectors) {
    const el = document.querySelector(sel);
    if (!el) continue;
    const src = el.src || el.getAttribute('data-src') || '';
    if (/\/galleries\/\d+\/\d+\.(jpg|png|webp)/i.test(src)) return src;
  }
  // Fallback: scan all imgs on the page
  for (const img of document.querySelectorAll('img')) {
    const src = img.src || img.getAttribute('data-src') || '';
    if (/galleries\/\d+\/\d+\.(jpg|jpeg|png|webp|gif)/i.test(src)) return src;
  }
  return null;
}
zipper.js — ZIP file structure (store mode, no compression)
text 
ZIP layout (per file):

  [ Local File Header  ]  30 + len(filename) bytes
  [ File Data          ]  raw Uint8Array
  [ Central Dir Entry  ]  46 + len(filename) bytes  (written at end)
  [ End of Central Dir ]  22 bytes

CRC-32 computed via 256-entry lookup table (standard polynomial 0xEDB88320).
Compression method = 0 (Store). No deflate. Compatible with all ZIP tools.
popup.js — download loop
javascript 
// base = "https://i3.nhentai.net/galleries/177013/"
let n = 1;
while (!cancelled) {
  const response = await fetch(`${base}${n}.${ext}`);

  if (!response.ok) break; // 404 = end of gallery

  const buffer = await response.arrayBuffer();
  zip.addFile(`${code}/${String(n).padStart(4,'0')}.${ext}`, new Uint8Array(buffer));
  n++;
}
// triggers browser download as {code}.zip
const blob = new Blob([zip.generate()], { type: 'application/zip' });
a.download = `${code}.zip`;

v1.1 — CDN Subdomain Fix

The extension was showing "only works on nhentai.net" even when already on nhentai.net. Root cause: the old code tried to parse the CDN URL from the browser's address bar. The tab URL is always nhentai.net/g/{id}/{page}/ with no image path in it — so parsing always failed.

Fix: inject scrapeImageSrc() into the live reader page using chrome.scripting.executeScript(). This reads the src from the <img> already rendered on screen, giving us the exact CDN URL — correct subdomain and all — without guessing.

Also cleaned up a duplicate host_permissions entry in manifest.json and removed a broken jszip.min.js placeholder that was fetching from a blocked CDN. The ZIP engine was already fully custom anyway.

⚠ Known: Pages that lazy-load images (scroll-triggered) may return null from the scraper if the first image hasn't rendered yet. Workaround: scroll to page 1 of the reader, let it fully load, then open the popup.

Initial release — OpenNH-Download v1.0.0

First working build. The popup UI is a dark-themed panel with a live progress bar, status log, and cancel button. popup.js handles URL parsing, the fetch loop, and ZIP trigger. zipper.js is a fully custom ZIP encoder — local file headers, central directory, CRC-32 table — no external libraries.

Chrome Manifest v3. Permissions locked to *.nhentai.net/*. Gallery images downloaded as 0001.jpg, 0002.jpg… inside a folder named after the gallery code, then packed into {code}.zip and downloaded via a Blob URL.
